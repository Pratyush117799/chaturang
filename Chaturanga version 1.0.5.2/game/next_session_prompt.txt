Chaturanga v1.0.4 is a fully functional web-based ancient 4-player
Indian chess game — the ancestor of all modern chess variants. It runs
entirely in the browser, no framework, vanilla HTML/CSS/JS only.

The game is culturally grounded in:
  · Sanskrit piece names (Ratha/Rook, Ashwa/Horse, Danti/Elephant,
    Nara/Pawn, Raja/King)
  · Dice-driven movement (Pāśaka: face 1=Ratha, 2/5=any, 3=Ashwa,
    4=Danti, 6=Nara or Raja)
  · Mahabharata and Bhagavad Gita as cultural backdrop
  · Acharya Drona as the in-game guide/narrator voice

Aesthetic: dark background (#0f0c07), gold accents (#c9a84c),
Cinzel (serif, titles) + Outfit (sans, body), Font Awesome icons.
Every cultural reference must be authentic — not decorative.

Piece movement rules (critical — validate every puzzle/board):
  Rook (dice 1):     slides orthogonal any distance, blocked by pieces
  Horse (dice 3):    L-shape (±1,±2)|(±2,±1), jumps over pieces
  Elephant (dice 4): exactly 2 squares diagonal ONLY, jumps over pieces
                     colour-bound — cannot change diagonal colour complex
  Pawn (dice 6):     1 forward, captures diagonal forward, promotes at
                     rank 8 (player 0/1) — promotes to Rook
  King (dice 6):     1 step any direction, cannot capture another King
  Dice 2/5:          any piece of player's choice

Capture rule: minor pieces (Nara/Ashwa/Danti) CANNOT capture other
minor pieces in the full game. Only Ratha and Raja can capture minors.
(Puzzle engine uses simplified rules — direct captures allowed in puzzles.)

Players: 0=Red (Team 1), 1=Blue (Team 2), 2=Green (Team 1), 3=Yellow (Team 2)
Pawns: players 0+2 move upward (rank increases), 1+3 move downward.

WHAT IT IS
──────────
One puzzle released every day at 6:00 AM IST (UTC+5:30). The same
puzzle is available to every player worldwide on that calendar day.
First correct solve = #1 on the global leaderboard. Scoring is based
on solve time + hints not used. A "Shubh streak" tracks consecutive
daily solves.

The feature is themed to the Hindu lunar calendar — puzzle difficulty
and type vary based on the tithi (lunar day):
  · Ekadashi (11th tithi)  — hardest puzzle of the month (ELO 700+)
  · Amavasya (new moon)    — endgame / king-hunt specialisation
  · Purnima (full moon)    — alliance/team combination puzzles
  · Regular days           — rotating through all puzzle types

Special puzzles release on major festivals:
  Diwali, Janmashtami, Holi, Makar Sankranti, Navratri Day 9.

ARCHITECTURE DECISION — NO SERVER REQUIRED
───────────────────────────────────────────
There is no backend. The leaderboard is simulated locally using
localStorage. Here is how the "global" leaderboard works without a
server:

  · The daily puzzle is selected deterministically from the 200-puzzle
    library using a seeded algorithm:
      dayIndex = Math.floor(Date.now() / 86400000)  ← changes at UTC midnight
      puzzleIndex = dayIndex % ChaturangaPuzzleData.length
      The IST offset (UTC+5:30 = 19800 seconds) is applied:
      dayIndex = Math.floor((Date.now() + 19800000) / 86400000)

  · This guarantees every player worldwide gets the same puzzle on the
    same calendar day (in IST). No server needed.

  · The leaderboard is seeded with 8 realistic fake competitors whose
    scores are deterministically generated from the same dayIndex seed.
    Real player scores are stored locally and shown alongside the fakes.
    This creates the illusion of a live leaderboard without a backend.
    Each fake competitor has a Sanskrit name, a city, and a realistic
    solve time (60–900 seconds, weighted by puzzle ELO).

  · When the player solves the puzzle, their result is saved to
    localStorage under key: chaturanga_shubh
    Format: { [dateString]: { solved, time, hintsUsed, score, streak } }

HINDU CALENDAR TITHI CALCULATION
──────────────────────────────────
The tithi (lunar day) cycles 1–30 approximately every 29.5 days.
Use this simplified calculation (good enough for theming — does not
need to be astronomically precise):

  const LUNAR_EPOCH = 1704067200000; // Jan 1 2024 00:00 UTC (known new moon)
  const LUNAR_CYCLE = 29.530588853 * 24 * 60 * 60 * 1000; // ms
  function getTithi(now = Date.now()) {
    const elapsed = ((now - LUNAR_EPOCH) % LUNAR_CYCLE + LUNAR_CYCLE) % LUNAR_CYCLE;
    return Math.floor(elapsed / LUNAR_CYCLE * 30) + 1; // 1–30
  }
  // tithi 11 or 26 = Ekadashi
  // tithi 15 = Purnima (full moon)
  // tithi 30 or 1 = Amavasya (new moon)

WHAT IT IS
──────────
The Akhara (Sanskrit: wrestling school/training ground) is a
drill-based training mode that builds board vision — the ability to
read a Chaturanga position instantly without calculating move by move.
Board vision is the single skill that separates fast improvers from
slow ones. Every drill is timed.

This is a completely separate page: akhara.html

  Yoddha (Warrior)     — 200–399              — sees one-move tactics
  Sarathi (Charioteer) — 400–599              — reads full board threats
  Arjuna               — 600–799              — spots combinations instantly

Akhara ELO is separate from puzzle ELO. It is stored under:

THE FOUR DRILLS
────────────────

DRILL 1: FLASH DRILL
  How it works:
    A random board position is shown for exactly 3 seconds.
    The board disappears. The player is asked 3 questions:
      Q1: "How many of your pieces are hanging (undefended + threatened)?"
         Answer is a number 0–8. Select via +/- buttons.
      Q2: "How many enemy pieces can you capture right now?"
         Answer is a number 0–8.
      Q3: "Which piece is most threatened?" (multiple choice: Ratha/Ashwa/
         Danti/Nara/Raja/None)
    Each correct answer = 10 points. Max 30 per position.
    10 positions per session. Total possible: 300 points.

  Board generation:
    Use 10 pre-built positions from puzzle-data.js boardStates (use
    the boardState from random puzzles in the 200-puzzle library).
    Do NOT randomly generate boards — pre-built positions guarantee
    legal, meaningful positions.

  Timing:
    Show board for 3000ms exactly. Use a visible countdown bar that
    depletes in real time (CSS animation, width 100%→0% in 3s).
    After 3s, hide the board div (opacity: 0, pointer-events: none)
    and show the question panel with a 15-second answer timer.

DRILL 2: FORK FINDER
  How it works:
    Show a board position. The player must click the single square
    where a Horse (Ashwa) of their colour could land to fork the
    enemy Raja AND at least one other enemy piece.
    Timer: 5 seconds visible countdown per position.
    Correct click within time = +20 points.
    Wrong click = 0 (no penalty, move to next).
    10 positions per session. Max 200 points.

  Board generation:
    Use only puzzles of type 'ashwa-fork' from puzzle-data.js.
    The fork square is the solution[0].to value.
    Highlight valid pieces (the two pieces being forked) in the
    question text only — NOT on the board.
    Legal horse move validation: from candidate square, check if the
    horse can reach both the Raja and at least one other enemy piece
    in (±1,±2)|(±2,±1).

  Visual feedback:
    On correct click: flash the square green for 500ms.
    On wrong click: flash the square red for 300ms.
    After time expires with no click: show correct square in gold.

DRILL 3: RATHA HUNT
  How it works:
    Show a board position. The player must click the enemy Ratha
    that can be captured in ONE move by any of their pieces.
    Timer: 8 seconds per position.
    Correct = +15 points. 10 positions. Max 150 points.

  Board generation:
    Use puzzles of type 'ratha-hunt' from puzzle-data.js.
    The target is always the piece at solution[0].to (the capture
    destination in the first solution step).
    Only show positions where exactly one enemy Ratha is capturable
    in one move (keeps the drill unambiguous).

  Visual feedback:
    Player clicks a square. If that square contains the capturable
    Ratha: green flash + next position.
    If wrong piece: red flash, no penalty, timer continues.
    On timeout: highlight correct Ratha in gold.

DRILL 4: DICE DRILL
  How it works:
    Show a board position AND a dice face value (1–6).
    The player must click the BEST move for that dice roll.
    "Best" is defined as: captures the highest-value piece available,
    or if no capture, moves toward the centre.
    Timer: 10 seconds per position.
    Correct = +25 points. 10 positions. Max 250 points.

  Board generation:
    Use any puzzle from puzzle-data.js where forcedDice[0] is defined.
    The correct move is solution[0] (from→to).
    The dice face shown is forcedDice[0].

  Validation:
    The player clicks a destination square. If from+to matches
    solution[0], it is correct.
    Allow selecting the piece first (click piece, then destination).
    Show piece selection highlight as in puzzles.html.

SCORING AND ELO UPDATE
───────────────────────
After a drill session (10 positions) completes:

  drillScore = total points earned / max possible points × 100 (percent)

  ELO adjustment per drill:
    if drillScore >= 80: eloAkhara += 25
    if drillScore >= 60: eloAkhara += 10
    if drillScore >= 40: eloAkhara += 0  (no change)
    if drillScore <  40: eloAkhara -= 10

  eloAkhara is bounded 0–1000.
  Rank is recomputed from eloAkhara after each session.
  Display the rank change with an animation if tier changes.
